# Chapter Module Source Map

Read these in this order unless the task is extremely narrow:

1. `docs/preproduction-map.md`
   Use when you need precedence between project docs.
2. `docs/action-rpg-terminology.md`
   Use for scene-role labels and gameplay vocabulary.
3. `docs/content-conventions.md`
   Use for IDs, scene metadata, quest metadata, and Markdown-vs-YAML policy.
4. `docs/planning/godot-modular-build-and-story-plan.md`
   Use for chapter-module architecture, build profiles, and validation expectations.
5. `docs/planning/task-tracker.md`
   Use to confirm current implementation priorities.
6. The relevant `docs/chapters/chapter-##-*.md`
   Use for narrative intent and scene order.
7. The relevant `game/data/chapters/`, `game/data/scenes/<chapter>/`, and `game/data/quests/<chapter>/`
   Use for mirrored runtime-facing content.

Prefer reading only the chapter you are actively editing instead of loading all chapter docs.
