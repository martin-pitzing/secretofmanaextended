# SOMR Visual Style Authoring

Use this skill when the task is about environment art direction, texture/material guidance, placeholder stills, chapter visual profiles, combat-benchmark look, or scene readability.

Read `references/source-map.md` first.

If the task is specifically about reviewing or defining deliverables, also read `references/visual-review-checklist.md`.

## Workflow

1. Anchor to the gameplay read first.
   Protect locomotion, attack commitment, telegraphs, combat lanes, and damage feedback before adding ornamental detail.

2. Stay inside the reference lane.
   Use `Secret of Mana` for party-forward sacred fantasy, `Alundra` for jump-capable traversal and dense room reads, and `Terranigma` for brisk movement and combat rhythm.

3. Start with reusable chapter visual rules.
   Build chapter visual profiles, prop families, texture/material libraries, and palette logic before scene-specific exceptions.

4. Add detail through function and history.
   Prefer material storytelling, ritual wear, and social use over random clutter or generic fantasy noise.

5. Keep placeholder art in the runtime pipeline.
   If final art is not ready, use story stages and still boards that can still be reviewed in-engine.

6. Follow Chapter 1 anchor order.
   The current visual production priority is `Water Palace`, then `Forbidden Falls`, then `Potos`.

7. Ask questions when the visual brief is vague.
   Present 2-3 options and recommend one when ambiguity affects mood, palette, prop density, texture direction, placeholder-vs-production level, or benchmark priority.

## Output Checklist

- the work still reads as a `polished 2D Mana-lineage action RPG`
- the scene is readable at gameplay distance
- palette, materials, and prop families are explicit
- texture direction supports region identity instead of generic clutter
- ARPG readability is protected before secondary decoration
- Godot-first dressing targets or placeholder-stage use are stated when relevant
