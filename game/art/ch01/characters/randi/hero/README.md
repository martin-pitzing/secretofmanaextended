# Randi Hero Atlas

This folder is the first `hero-spec` locomotion lane for Randi.

Current decisions:

- `64x64` frame canvas
- `hero64_tall` is the primary runtime-facing atlas target
- `native32_tall` is for comparison only
- `4-direction + side flip`
- ears stay a `subtle accent`
- running is triggered by `double tap in one direction`
- jump is planned as `combat-usable` once the jump layer is implemented

Current runtime scope:

- `idle_down`, `idle_up`, `idle_side`
- `walk_down`, `walk_up`, `walk_side`
- `run_down`, `run_up`, `run_side`
- `attack_down`
- `attack_side`
- `attack_up`
- `jump_down`, `jump_side`, `jump_up`
- `hit_down`

The current files are generated by:

```powershell
.\tools\generate-randi-hero-sprites.ps1
```

Manifest-derived runtime refresh:

```powershell
.\tools\export-randi-hero-runtime-frames.ps1 -BuildContactSheet
```

Current source split:

- all current hero runtime clips are now exported from the `hero64_tall` transfer manifest
- `idle_up`, `walk_down`, `walk_up`, `run_down`, and `run_up` are no longer placeholder atlas frames
- those `up` and `down` locomotion loops are still bridge-quality source transfers using turnaround poses plus per-frame offsets, not final authored locomotion sheets
- `jump_down` and `jump_up` now come from the same manifest lane and are wired into runtime preview playback
- `hit_down` is wired into runtime, with directional fallback still resolving to the down-facing reaction until side/up hit variants are authored

This is the current runtime-facing hero lane for sandbox validation.
It is not the final production-quality hero sheet yet.

- add authored side/up hit reactions instead of down-facing fallback
